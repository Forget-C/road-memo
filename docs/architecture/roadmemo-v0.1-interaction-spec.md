# RoadMemo V0.1 页面交互说明

## 1. 文档目标

这份文档用于明确 RoadMemo V0.1 的页面职责、核心交互路径、关键状态与页面之间的跳转关系。

配套基线：

- 数据模型以 `roadmemo-v0.1-data-model.md` 为准
- 产品冻结范围以 `roadmemo-v0.1-freeze-checklist.md` 为准
- 视觉方向以 `roadmemo-v0.1-visual-baseline.md` 为准

## 2. 全局结构

底部导航固定为：

1. 首页
2. 记录
3. 统计
4. 设置

全局规则：

- 首页默认展示当前默认车辆
- 页面切换不丢失当前车辆上下文
- 新增记录时默认带入当前车辆
- 若用户切换车辆，首页、记录、统计、续期相关内容同步刷新

## 3. 首页

### 3.1 页面目标

首页负责回答 4 个问题：

1. 这辆车这个月总共花了多少钱
2. 支出主要分布在哪几类
3. 最近一次关键记录是什么
4. 有没有快到期、需要处理的事项

### 3.2 页面结构

从上到下建议为：

1. 当前车辆头部
2. 本月总支出主卡
3. 分类摘要区
4. 最近记录区
5. 即将到期提醒区
6. 快捷新增入口

### 3.3 关键模块说明

#### 当前车辆头部

展示：

- 车辆品牌 + 车型
- 车牌号或昵称
- 切换车辆入口

交互：

- 点击头部进入车辆切换浮层或车辆列表页
- 切换后整页数据刷新

#### 本月总支出主卡

展示：

- 本月总支出金额
- 同比/环比先不做
- 本月时间范围说明

交互：

- 点击进入统计页默认“本月”视图

#### 分类摘要区

展示 4 类摘要：

- 能源
- 保养
- 费用
- 续期

交互：

- 点击某一类摘要，跳转到记录页并定位到对应 Tab

#### 最近记录区

展示：

- 最近一条能源记录
- 最近一条保养记录

交互：

- 点击进入详情页或编辑页

#### 即将到期提醒区

展示：

- 最近到期的 1 到 3 条事项
- 到期剩余时间
- 状态标签

交互：

- 点击进入续期详情或提醒详情
- 若无提醒，则显示“暂无即将到期事项”

#### 快捷新增入口

建议使用悬浮按钮：

- 点击后弹出快捷面板
- 可直接选择：
  - 新增能源
  - 新增保养
  - 新增费用
  - 新增续期

### 3.4 首页状态

#### 首次使用

展示：

- 空状态说明
- 添加第一辆车按钮
- 添加首条记录引导

#### 无记录但已有车辆

展示：

- 本月总支出为 0
- 引导新增第一条记录

#### 有提醒但无记录

允许存在，不冲突。

## 4. 记录页

### 4.1 页面目标

记录页负责：

- 按类型查看历史记录
- 快速新增
- 查找和编辑已有记录

### 4.2 一级入口

记录页顶部 Tab 固定为：

1. 能源
2. 保养
3. 费用
4. 续期

### 4.3 顶部区域

建议包含：

- 当前车辆信息
- Tab 切换
- 筛选入口
- 搜索入口

V0.1 建议：

- 保留基础筛选
- 搜索可以延后到 V1.1

### 4.4 各 Tab 的列表结构

#### 能源

每条展示：

- 日期时间
- 能源类型
- 数量
- 金额
- 里程
- 是否补满

交互：

- 点击进入能源记录详情 / 编辑
- 长按或滑动显示删除

#### 保养

每条展示：

- 日期
- 保养类型
- 金额
- 里程
- 是否设置下次保养

#### 费用

每条展示：

- 日期
- 分类
- 金额
- 备注摘要

#### 续期

每条展示：

- 类型
- 到期日期
- 金额
- 是否开启提醒
- 当前状态

### 4.5 新增入口逻辑

记录页右下角悬浮按钮逻辑：

- 当前在哪个 Tab，点击默认新增该类型记录
- 长按或点击展开可切换到其他类型新增

### 4.6 编辑与删除

统一规则：

- 点击进入编辑页
- 删除前弹确认
- 删除后列表刷新

特殊规则：

- 删除能源记录时，如影响能耗区间，删除后自动重算
- 删除续期记录时，同步删除来源提醒

## 5. 新增 / 编辑记录页

## 5.1 页面目标

统一承载 4 类记录的新增和编辑：

1. 能源
2. 保养
3. 费用
4. 续期

### 5.2 页面结构

统一结构建议：

1. 顶部标题
2. 车辆选择
3. 核心字段表单
4. 可选字段折叠区
5. 保存按钮

### 5.3 能源表单

核心字段：

- 车辆
- 日期时间
- 能源类型
- 里程
- 数量
- 总金额
- 是否补满

条件字段：

- 燃油时显示：油号、加油站
- 充电时显示：充电方式、充电站

交互规则：

- 切换能源类型时，动态切换字段
- 金额与数量填写后可实时反算单价展示
- 保存前做里程递增校验

### 5.4 保养表单

核心字段：

- 车辆
- 日期
- 保养类型
- 金额

可选字段：

- 里程
- 门店
- 备注
- 下次保养日期
- 下次保养里程

交互规则：

- 若填写下次保养日期或里程，可自动勾选创建提醒

### 5.5 费用表单

核心字段：

- 车辆
- 日期
- 分类
- 金额

可选字段：

- 备注

### 5.6 续期表单

核心字段：

- 车辆
- 类型
- 金额
- 到期日期

可选字段：

- 生效日期
- 机构名称
- 单据编号
- 备注
- 是否开启提醒

交互规则：

- 开启提醒时，显示“提前 N 天提醒”设置
- 保存后自动维护来源提醒

### 5.7 保存结果

保存成功后：

- 返回来源页
- 列表刷新
- 首页摘要刷新
- 如涉及提醒，触发提醒同步逻辑

## 6. 统计页

### 6.1 页面目标

统计页负责把“记录”转换成“理解”。

V0.1 只聚焦以下问题：

1. 本月 / 本年的总支出趋势如何
2. 钱主要花在哪
3. 多辆车谁更费钱
4. 最近能耗表现如何

### 6.2 页面结构

建议从上到下为：

1. 时间范围切换
2. 总支出趋势
3. 分类占比
4. 单车对比
5. 能耗趋势

### 6.3 时间范围切换

支持：

- 本月
- 本年
- 自定义

V0.1 可先做：

- 本月
- 本年

自定义可视时间情况后置。

### 6.4 图表交互

#### 趋势图

- 点击数据点显示具体金额
- 支持轻量悬浮信息

#### 分类图

- 点击某分类高亮并显示占比
- 点击后可跳到记录页对应分类

#### 单车对比

- 若只有 1 辆车，显示提示而不是空图

#### 能耗趋势

- 燃油车显示油耗
- 纯电车显示电耗
- 插混车同时显示两条趋势

## 7. 续期与提醒逻辑

### 7.1 入口位置

V0.1 建议通过两条路径进入：

1. 记录页中的“续期”Tab
2. 首页中的即将到期提醒区

如后续需要，也可以扩展为独立提醒页。

### 7.2 续期项状态认知

用户在这一块最关心的是：

- 什么时候到期
- 是否已经开启提醒
- 是否已经处理

因此列表项优先展示：

- 类型
- 到期日期
- 剩余时间
- 提醒状态

### 7.3 提醒状态流转

建议 UI 层体现以下状态：

- 待提醒
- 已触发
- 已处理
- 已忽略

交互：

- 用户可以从提醒卡片直接标记“已处理”
- 用户可以临时关闭提醒
- 删除续期事项时，同步删除来源提醒

## 8. 设置页

### 8.1 页面目标

设置页负责：

- 个性化偏好设置
- 数据导出
- 备份恢复
- 主题与单位设置

### 8.2 建议模块

1. 显示与主题
2. 单位与货币
3. 提醒偏好
4. 数据管理
5. 关于应用

### 8.3 数据管理

应包含：

- CSV 导出
- 本地备份
- 本地恢复

交互重点：

- 恢复前必须二次确认
- 明确提示“将覆盖当前本地数据”

## 9. 关键用户路径

### 9.1 首次使用路径

1. 打开 App
2. 进入引导
3. 添加第一辆车
4. 进入首页
5. 添加第一条记录

### 9.2 日常记账路径

1. 从首页点击新增
2. 选择记录类型
3. 填写表单
4. 保存
5. 返回首页或记录页查看变化

### 9.3 续期处理路径

1. 首页看到即将到期提醒
2. 点击进入续期详情
3. 补充或修改到期信息
4. 开启提醒
5. 到期前收到通知

### 9.4 多车切换路径

1. 点击首页车辆头部
2. 切换车辆
3. 全局统计和记录同步刷新

## 10. 关键交互原则

### 10.1 少跳转

- 尽量在当前场景完成操作
- 新增成功后优先返回来源页

### 10.2 强反馈

- 保存后立即看到首页或列表变化
- 删除后立即刷新数据

### 10.3 表单低负担

- 核心字段优先
- 可选字段折叠
- 动态表单减少无关输入

### 10.4 结构一致

- 所有列表页遵循统一信息层级
- 所有编辑页遵循统一保存逻辑

## 11. 视觉与交互协同要求

后续设计或实现时，需同时满足：

- 扁平化
- 干净整洁
- 留白充足
- 卡片极简
- 弱阴影
- 强信息层级

不允许回到：

- 仪表盘感过强
- 车机感
- 拟物卡片堆叠
- 装饰多于信息
