# RoadMemo V0.1 视觉基线

## 1. 冻结结论

RoadMemo V0.1 的视觉方向冻结为：

- 扁平化
- 干净整洁
- 轻量留白
- 现代商业软件风格
- 不走传统车机仪表盘风格

一句话定义：

> 更像成熟移动商业软件，而不是厚重的汽车工具面板。

## 2. 视觉气质

目标气质：

- 清爽
- 理性
- 可信
- 高效
- 克制

避免：

- 老气
- 厚重阴影
- 强拟物
- 金属仪表盘感
- 霓虹感
- 复杂渐变

## 3. 配色方向

主配色冻结为：

- 背景：暖白、浅灰白
- 卡片：纯白
- 主文字：深灰 / 炭黑
- 次文字：中灰
- 主强调色：柔和 teal
- 警示色：克制橙色
- 图表辅助色：浅蓝灰 / 淡青色

明确约束：

- 避免紫色主导
- 避免大面积高饱和颜色
- 避免深色沉重背景作为默认主题

## 4. 组件风格

### 4.1 卡片

- 卡片尽量扁平
- 阴影极弱，必要时才使用
- 圆角中小尺度，不做过度圆润
- 主要通过留白和层级区分信息，而不是靠厚重容器

### 4.2 分隔方式

- 优先使用留白和细分隔线
- 少用强边框
- 少用高对比背景块

### 4.3 图标与插图

- 图标线性、简洁
- 插图只作辅助，不抢信息主层级
- 首页车辆图若保留，应弱化存在感

## 5. 页面风格要求

### 首页

- 一张核心总览卡作为视觉中心
- 次级数据用轻量信息块呈现
- 最近记录和提醒区块保持干净，不堆装饰

### 记录页

- 更接近商业软件列表
- 强调扫描效率
- 列表项统一结构，不做厚卡片堆叠

### 统计页

- 图表轻量化
- 强调可读性，不追求花哨
- 留白足够，避免“满屏都是图”

### 续期页

- 强调状态与到期时间
- 使用简洁状态标签与开关
- 让用户一眼判断“是否需要处理”

## 6. 实现侧建议

后续进入 UI 设计和 Compose 实现时，建议遵循：

- 优先用排版层级建立信息主次
- 颜色只承担辅助强调，不承担主要结构职责
- 卡片背景尽量统一
- 页面默认保持浅色主题优先

## 7. 与产品结构的关系

这套视觉基线特别适合当前冻结版产品结构：

- 首页
- 记录
- 统计
- 续期

因为这几个页面都偏信息型界面，适合：

- 低噪音布局
- 扁平化组件
- 商业软件式层级

## 8. 后续设计输出要求

后续若继续生成设计图或实现 UI，统一按以下关键词执行：

- flat
- clean
- light
- whitespace
- premium commercial mobile app
- minimal shadow
- calm teal accent
- no dashboard style

## 9. 当前结论

V0.1 后续所有视觉方案默认以本文件为准。  
若没有新的产品决策，不再回到“厚重、老气、仪表盘感强”的方向。
