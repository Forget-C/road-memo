# RoadMemo V0.1 视觉参考映射表

## 1. 目标

为 `长路有记 RoadMemo` 的各核心页面指定明确的视觉参考来源，避免后续 UI 落地时风格漂移。

本映射表基于以下设计方向组合：

- 基础框架：`Linear`
- 财务表达：`Wise / Revolut`
- 车辆气质：`Tesla`

参考来源：

- GitHub: https://github.com/VoltAgent/awesome-design-md/tree/main/design-md
- Linear: https://getdesign.md/linear.app/design-md
- Revolut: https://getdesign.md/revolut/design-md
- Wise: https://getdesign.md/wise/design-md
- Tesla: https://getdesign.md/tesla/design-md

---

## 2. 总体映射

### 2.1 全局框架

主参考：`Linear`

用于定义：

- 页面留白
- 卡片层级
- 顶部标题区
- 标签页样式
- 列表密度
- 轻量状态标签

落地原则：

- 保持浅色背景
- 卡片扁平，弱阴影
- 重点靠字号、字重、留白，不靠重装饰
- 列表信息高密但不拥挤

---

## 3. 页面映射

### 3.1 首页

主参考：`Linear + Revolut + Tesla`

分工建议：

- `Linear`
  - 页面整体结构
  - 模块分区方式
  - 卡片排布节奏
- `Revolut`
  - 本月总支出大数字
  - 分类摘要卡
  - 数据模块的“财务产品感”
- `Tesla`
  - 当前车辆卡片气质
  - 车辆信息区域的克制高级感

落地重点：

- 首页顶部车辆模块只借 `Tesla` 的气质，不做车机风
- 首页主卡突出“本月总支出”
- 次级信息按 `能源 / 保养 / 费用 / 续期` 平铺拆开

不建议：

- 使用重渐变
- 做大面积深色背景
- 做拟物仪表盘

---

### 3.2 记录页

主参考：`Linear`

用于定义：

- 一级 Tab
- 列表扫描效率
- 卡片操作按钮布局
- 编辑 / 删除的轻操作感

落地重点：

- 列表项优先强调“信息扫描效率”
- 每条记录控制在 2 到 3 层信息
- 操作按钮弱化但可快速触达

适合借鉴的具体感觉：

- 简洁标题
- 紧凑列表
- 轻量操作区

---

### 3.3 新增 / 编辑表单页

主参考：`Wise + Linear`

分工建议：

- `Wise`
  - 表单亲和感
  - 输入流程的低压力体验
  - 成功 / 失败反馈的温和表达
- `Linear`
  - 表单分组
  - 辅助信息层级
  - 页面节奏和模块边界

落地重点：

- 输入框尽量简洁
- 校验提示用明确短句
- 错误用颜色提醒，但不要做大面积红色警示
- 保存按钮状态明显

---

### 3.4 统计页

主参考：`Revolut + Wise`

分工建议：

- `Revolut`
  - 总支出主卡
  - 分类汇总表现
  - 财务工具的专业感
- `Wise`
  - 图表和说明文字的克制关系
  - 数据解释区的可读性

落地重点：

- 图表先轻量，不做炫技
- 先强调“读懂统计”，再强调“好看”
- 颜色使用克制，图表不要出现太多杂色

---

### 3.5 提醒页

主参考：`Wise + Linear`

用于定义：

- 到期事项列表
- 提醒状态操作
- 来源跳转按钮

落地重点：

- 强调“还有多久到期”与“下一步动作”
- `已处理 / 忽略 / 查看来源` 要有清楚优先级
- 不做沉重警报风，保持工具型冷静表达

---

### 3.6 设置页

主参考：`Wise`

用于定义：

- 数据管理区
- 备份恢复
- 导出动作
- 偏好设置

落地重点：

- 比首页更轻、更安静
- 危险动作靠确认弹窗，不靠视觉恐吓
- 反馈信息简洁明确

---

### 3.7 车辆管理页

主参考：`Tesla + Linear`

分工建议：

- `Tesla`
  - 车辆信息气质
  - 顶部车辆卡的品牌感
- `Linear`
  - 多车列表组织
  - 默认车辆标记
  - 编辑操作密度

落地重点：

- 车辆卡要有一点“车产品气质”
- 但整体仍是效率工具，不是展示型相册

---

## 4. 组件级参考

### 4.1 顶部标题区

参考：`Linear`

规则：

- 标题短
- 副标题少
- 不堆叠多行说明

### 4.2 金额主卡

参考：`Revolut`

规则：

- 大字号
- 强对比
- 只突出一个核心数字

### 4.3 输入表单

参考：`Wise`

规则：

- 统一输入框高度
- 统一错误提示位置
- 主要 CTA 明确

### 4.4 车辆信息卡

参考：`Tesla`

规则：

- 极简
- 高级灰度感
- 轻品牌气质

### 4.5 操作列表

参考：`Linear`

规则：

- 按钮不喧宾夺主
- 信息优先，操作次之

## 5. 配色落地建议

基于现有视觉基线，延续以下方向：

- 主背景：暖白或极浅灰
- 卡片背景：纯白
- 主文字：深灰
- 次文字：中灰
- 主强调色：柔和 teal
- 风险 / 到期：橙色
- 错误：克制红色

颜色使用原则：

- 一个页面只允许 1 个主强调色
- 风险色只用于到期、错误、危险确认
- 图表色不超过 4 种

---

## 6. 不建议借鉴的方向

以下风格不适合当前阶段的 RoadMemo：

- 重拟物
- 深色大面积车机风
- 赛博科技风
- 过度渐变
- 装饰先于信息的展示型风格

原因：

- 会削弱工具产品的可信度
- 会影响录入和扫描效率
- 不符合当前已冻结的浅色扁平基线

---

## 7. 最终结论

RoadMemo V0.1 的最优视觉参考组合为：

- 全局结构：`Linear`
- 财务表达：`Revolut / Wise`
- 车辆气质：`Tesla`

更具体地说：

- 首页：`Linear + Revolut + Tesla`
- 记录页：`Linear`
- 表单页：`Wise + Linear`
- 统计页：`Revolut + Wise`
- 提醒页：`Wise + Linear`
- 设置页：`Wise`
- 车辆页：`Tesla + Linear`

这套组合既能保持：

- 扁平化
- 干净整洁
- 商业软件感

也能保留 RoadMemo 作为“车辆成本工具”的产品气质。
