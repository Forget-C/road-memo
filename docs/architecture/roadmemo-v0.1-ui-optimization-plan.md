# RoadMemo V0.1 UI 优化方案

## 1. 目标

参考商业化车辆账单软件的专业感、数据表达力和页面完成度，
但不直接复制其视觉语言。RoadMemo 需要形成自己的产品风格：

- 更克制
- 更清爽
- 更现代
- 更像工具型商业软件
- 保留一点车辆产品应有的“机械感 / 仪表感”，但不过度车机化

一句话总结：

**借它的专业感，不借它的造型。**

## 2. 从参考图里应该学什么

这张图最值得借鉴的，不是某个具体卡片形状，而是它的这些优点：

### 2.1 首页有明确主角

它把“最新油耗”放到了最强位置，用户一眼就知道：

- 当前最重要的数据是什么
- 这辆车最近表现如何

对 RoadMemo 来说，这种思路要保留：

- 首页必须有一个主卡
- 主卡只回答一个最重要问题
- 其余统计降级为辅助信息

### 2.2 数据卡比列表更有产品感

参考图里不是单纯把数字堆成列表，而是把：

- 关键指标
- 时间范围
- 趋势图

都放进了结构完整的内容卡里。

这会让产品显得：

- 完整
- 可被信任
- 不是临时拼出来的 demo

### 2.3 顶部区域有品牌气氛

蓝色大背景 + 半透明氛围层，让顶部有明显“品牌区”。

这不一定要照搬成蓝色，但 RoadMemo 也需要一个：

- 有识别度的首页头部区
- 不只是白底文字

### 2.4 图表和数据是联动的

参考图不是“只有一张图”，而是：

- 上面有摘要
- 中间有统计
- 下面有趋势

这形成了自然的数据叙事：

**当前表现 → 核心指标 → 趋势变化**

这正是 RoadMemo 统计页应该学的。

## 3. 不应该照抄的部分

为了避免像“换皮复刻”，这些点不建议直接采用：

### 3.1 不直接照抄高饱和蓝色大背景

这张图偏传统国产工具产品的“亮蓝渐变”路线，识别度强，
但也容易显得：

- 偏旧
- 偏重
- 不够国际化

RoadMemo 更适合：

- 低饱和冷青绿
- 浅雾蓝灰
- 暖白底

### 3.2 不照抄异形卡片

截图里顶部白卡有明显异形切角和拼接，这类设计很抓眼，
但风险也高：

- 开发成本高
- 容易过时
- 一旦别的页面跟不上，就会显得只有首页特别花

RoadMemo 应该更统一：

- 规则圆角
- 扁平卡片
- 轻阴影或几乎无阴影

### 3.3 不照抄“功能入口很多”的首页

截图底部和顶部信息密度很高，更偏成熟商业产品的功能扩张期。

RoadMemo 当前还是 V0.1：

- 首页应该更聚焦
- 少即是多

## 4. RoadMemo 的独立视觉方向

### 4.1 关键词

- 扁平化
- 干净整洁
- 可信
- 数据导向
- 有一点车辆属性，但不车机化

### 4.2 视觉气质

RoadMemo 建议采用：

- **金融工具的秩序感**
- **车辆产品的轻机械感**
- **效率软件的克制表达**

组合来源：

- 信息层级参考 `Linear`
- 数据模块参考 `Wise / Revolut`
- 车辆身份区参考少量 `Tesla`

### 4.3 配色建议

不要直接沿用参考图的高亮蓝，而是建立 RoadMemo 自己的色板：

- 主色：`冷青绿 / Teal`
- 背景：`暖白偏灰`
- 辅助底色：`浅雾蓝灰`
- 强调色：`深海军蓝`
- 警示色：`橙色`
- 成功色：`柔和绿色`

建议色板：

- `Primary`: `#178A84`
- `Primary Container`: `#DDF3F0`
- `Accent Blue`: `#7EB7E8`
- `Background`: `#F6F7F5`
- `Surface`: `#FFFFFF`
- `Surface Variant`: `#EEF2F1`
- `Warning`: `#F28A3B`
- `Text Primary`: `#18201F`
- `Text Secondary`: `#5F6A68`

## 5. 页面级优化方案

## 5.1 首页

### 当前问题

- 已经比早期版本好很多，但还偏“数据卡拼装”
- 头部区域不够有识别度
- 主卡不够强
- 快捷入口仍偏功能导向，而非数据导向

### 优化方向

首页改成三段式：

1. **品牌头部区**
2. **本月主卡**
3. **数据摘要与最近动态**

### 推荐结构

```text
┌────────────────────────────┐
│ 顶部品牌头部区              │
│ 当前车辆 / 切车 / 提醒入口   │
│ 车辆副标题 / 时间范围        │
├────────────────────────────┤
│ 本月总支出主卡              │
│ 金额 + 环比说明 + 快捷动作   │
├────────────────────────────┤
│ 4 个摘要卡                  │
│ 能源 / 保养 / 费用 / 续期    │
├────────────────────────────┤
│ 最近记录                    │
├────────────────────────────┤
│ 即将到期提醒                │
└────────────────────────────┘
```

### 具体建议

- 顶部区域不要纯白，改成浅色品牌头部
- 主卡强调“本月总支出”，副指标显示本月能源占比或最近记录
- 快捷新增入口从“按钮排排坐”改成更轻的圆角 action pill
- 最近记录只保留 2 条，避免首页太长
- 提醒区做成更明确的状态卡，而不是简单按钮

## 5.2 记录页

### 当前问题

- 已经更规整了，但卡片还是偏列表程序感
- 缺少“信息优先级”的强化

### 优化方向

把记录页做成更像：

- 银行流水
- 账单列表
- 专业工具日志

### 列表卡建议

每条记录卡统一分成三层：

1. 顶部：类型标签 + 日期
2. 中间：主信息
3. 底部：金额 / 操作按钮

### 具体建议

- 每条记录加小型类型标签，不用高饱和
- 金额永远放在最醒目的右侧
- 删除按钮默认弱化，编辑为主动作
- 同一列表里不要让每条卡都像一个独立大面板，可适当降低卡片存在感

## 5.3 统计页

### 当前问题

- 信息结构已经有了
- 但图表表现还更像“可视化占位”

### 优化方向

统计页应该成为最像“商业产品”的页面。

建议结构：

1. 顶部总览主卡
2. 月度趋势
3. 分类拆分
4. 最近表现

### 具体建议

- 总览卡保持大数字
- 趋势区从“迷你卡”逐步升级为真正简化柱状图 / 折线图
- 分类区加入更正式的进度条或环形占比表达
- 最近表现每条不要写太长，控制成一句摘要

## 5.4 表单页

### 当前状态

已经统一成：

- 标题说明
- 当前车辆
- 主信息
- 辅助信息
- 保存区

### 下一步建议

- 增加输入提示文案层级
- 对金额、日期、里程这类字段加更弱的 hint
- 输入间距再统一一点
- Tab 的视觉再轻一点，减少“开发控件感”

## 5.5 车辆页

### 当前状态

方向已经对了，但还可以更“车辆产品”一点。

### 建议

- 默认车辆卡更像“当前座驾卡”
- 增加车型副描述区
- 动力类型标签可做成更轻的 badge
- 把“设为默认”从普通按钮做成更明确的主次状态切换

## 5.6 提醒页

### 当前状态

逻辑完整，但情绪表达偏弱。

### 建议

- 不同提醒类型有轻量视觉区分
- 到期状态更明确：
  - 即将到期
  - 今日到期
  - 已处理
- “查看来源 / 忽略 / 已处理” 三个动作层级继续拉开

## 5.7 设置页

### 当前状态

已经比之前整洁，但还更像配置页。

### 建议

- 设置页顶部继续强化“数据安全感”
- 数据管理区做成独立主模块
- “导出 / 备份 / 恢复” 做成统一动作组
- 结果反馈可以后续升级成 Snackbar + 状态条

## 6. 组件级统一规范

为了让整个产品摆脱 demo 感，组件必须统一。

### 6.1 卡片

- 默认圆角：`24dp`
- 大卡：`28dp`
- 阴影极轻，优先依赖背景对比
- 避免厚重描边

### 6.2 按钮

- 主按钮：纯色填充
- 次按钮：`FilledTonalButton`
- 危险操作：文本按钮或浅警示色

### 6.3 标签

- 用于类型、动力形式、默认状态
- 颜色弱，不要抢金额和主数据

### 6.4 数字

- 大数字单独强化
- 单位永远降一级
- 同页数字字号规则必须统一

### 6.5 图标

- 尽量用线性图标
- 风格统一，不混实体图标和线性图标

## 7. 实施优先级

### 第一阶段：高收益快速优化

- 首页头部区和主卡重做
- 记录卡信息层级统一
- 统计页图表卡升级

### 第二阶段：细节提升

- 提醒状态视觉优化
- 设置页动作组统一
- 车辆页身份感增强

### 第三阶段：设计系统沉淀

- 统一 spacing token
- 统一 card token
- 统一 button / badge / metric card 组件
- 补完整 UI 设计规范页

## 8. 对当前工程的落地建议

结合当前代码，我建议下一步不要零散修页面，而是按这 3 步推进：

### 8.1 先抽 UI tokens

在 `ui/theme` 和 `ui/components` 里补：

- spacing
- radius
- card variants
- badge variants
- metric card

### 8.2 先改首页

首页是最能拉开“demo 感”和“产品感”的页面。

### 8.3 再改统计页

统计页是最能体现“专业度”和“可信度”的页面。

## 9. 最终设计结论

这张商业产品截图对 RoadMemo 最有价值的启发，不是：

- 蓝色背景
- 异形卡
- 某个图表细节

而是：

- 它知道什么信息应该最大
- 它把数据组织成了完整产品页面
- 它让“车主工具”看起来像正式商业软件

RoadMemo 接下来应该做的是：

**保留我们已经建立起来的清爽扁平方向，再引入更强的首页主卡、统计层次和车辆身份感。**

这样最终出来的产品会：

- 比当前更专业
- 比参考图更克制
- 比普通 demo 更完整
- 同时保留自己的气质
